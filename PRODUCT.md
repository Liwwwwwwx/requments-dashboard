# Product

## Register

product

## Users

Developers, AI agents, and project coordinators managing cross-functional requirements across multiple projects. They use the board daily to track progress, filter noise, and decide what to work on next.

## Product Purpose

A single, dense dashboard that centralizes requirements, sub-tasks, interface contracts, and acceptance criteria across projects. It replaces scattered Markdown notes and manual status updates with an append-only event stream that always reflects the current state.

## Brand Personality

Functional, calm, precise. The interface should feel like a well-organized workshop: everything within reach, no unnecessary flourish.

## Anti-references

- Marketing dashboards with hero metrics and gradient text
- Overly rounded cards or pill-shaped containers on dense data
- Decorative motion that doesn't convey state
- Side-stripe colored borders on list items

## Design Principles

1. **Density serves speed.** Information-dense surfaces are preferable to sparse, scroll-heavy layouts.
2. **State over decoration.** Color and motion should communicate status, selection, or feedback, not brand personality.
3. **Consistency across projects.** The same visual vocabulary applies whether managing one project or ten.
4. **Hierarchy through spacing and type.** Avoid relying on borders alone to group information.

## Accessibility & Inclusion

- Target WCAG 2.1 AA
- Keyboard-navigable lists, filters, and drawers
- Respect `prefers-reduced-motion`
